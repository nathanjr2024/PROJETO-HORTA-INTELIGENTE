# Como Trabalhamos

## Ciclo de Desenvolvimento

Trabalhamos em **sprints semanais**. No início de cada semana definimos o que vai ser feito. No final, o que foi combinado deve estar funcionando.

---

## Quando uma tarefa pode ser iniciada (DoR)

1. Todo mundo da equipe entendeu o que precisa ser feito.
2. Já sabemos quais partes do código ou do hardware essa tarefa vai mexer.
3. A tarefa cabe dentro de uma semana de trabalho.

---

## Quando uma tarefa está pronta (DoD)

1. O código roda sem erros.
2. Foi testado manualmente e funcionou como esperado.
3. Não quebrou nada que já estava funcionando antes.

---

## Pull Requests

- Cada funcionalidade nova deve ser feita em uma branch separada (ex: `feat/irrigacao-automatica`).
- Antes de juntar com a branch `main`, pelo menos um outro membro da equipe deve revisar.
