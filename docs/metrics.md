# Como Medimos Nossa Qualidade

## 1. Tempo de Entrega

- **O que é:** Quantos dias levamos para concluir uma tarefa, do início até o merge no `main`.
- **Como medir:** Diferença entre a data de criação da branch e a data de aprovação do PR no GitHub.
- **Objetivo:** Concluir cada tarefa em no máximo **7 dias**.

## 2. Taxa de Erros

- **O que é:** Quantas vezes um código novo quebrou algo que já funcionava.
- **Como medir:** Anotar no backlog cada bug encontrado logo após um merge.
- **Objetivo:** Manter abaixo de **10% das entregas** com problema por sprint.

## 3. Disponibilidade do Site

- **O que é:** Quanto tempo o sistema fica fora do ar durante uma semana.
- **Como medir:** Checar manualmente ou usar uma ferramenta gratuita como UptimeRobot.
- **Objetivo:** O sistema deve ficar fora do ar por no máximo **5 horas por semana**.
